# tools/claude/specs/

RFC-style specifications — detailed, opinionated standards meant to be referenced across a team.

## Contents

- `ci-cd-specification.md` — comprehensive CI/CD standards (pipeline structure, security, testing, artifacts, branching).

## When to read

- Writing a team-wide CI/CD standard document.
- Looking for a reference to link from `CONTRIBUTING.md` or a runbook.

## vs `../guides/`

Guides are instructional. Specs are normative and long-form.
